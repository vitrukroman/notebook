## Divisor Analysis (CSES 2182) https://cses.fi/problemset/task/2182
**Дата:** 2024-02-20
**Тема:** мультиплікативні функції, мала теорема Ферма

**Ключові ідеї:**
- кількість дільників = добуток (ki+1)
- сума = добуток геом. прогресій
- добуток: показник p_i = k_i * D / 2, де D = кількість дільників
- показники беруться по mod-1 (мала т. Ферма)
- ділення на 2 по непростому модулю: ділимо того хто парний до взяття mod

**Де затупив:**
- думав що сума, а не добуток геом. прогресій
- ділення на 2 по mod(10^9+6)

**Патерн для майбутнього:**
- префіксні/суфіксні добутки для "всі крім i"
- якщо добуток парний — шукай хто парний і ділиш його


## Snakes (USACO Gold) https://usaco.org/index.php?page=viewproblem2&cpid=945
**Дата:** 2026-02-22  
**Тема:** DP, prefix sums, running max

**Ключові ідеї:**
- Для фіксованого сегмента `l..r` оптимальний розмір сітки `s` **не вибирається**, а **визначається умовою**: `s = max(d[l..r])` (інваріант сегмента)
- Вартість сегмента: `cost(l,r) = s * len - sum(l,r)`, де `sum(l,r)` через prefix sums

**Де помилився в мисленні:**
- Ти “попав” рівно там, де часто ламається мислення: ти намагався оптимізувати всередині сегмента через перебір `s`, хоча `s` не є незалежним вибором — він інваріант сегмента (дорівнює максимуму на відрізку). Тобто ти додав третю вісь, яка насправді визначається даними, а не “керується рішенням”.

**Патерн для майбутнього:**
- Якщо параметр має **покрити всі елементи сегмента** (`s ≥ a[i]`), то оптимум часто `s = max` → **не додавай його як стан/перебір**
- Для `max(l..r)` у сегментному DP: фіксуй `l` і розширюй `r`, тримаючи running max: `mx = max(mx, a[r])`
