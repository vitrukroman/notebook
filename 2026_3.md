## Stamp Painting (USACO Gold) https://usaco.org/index.php?cpid=791&page=viewproblem2
Дата: 2026-03-03  
Тема: DP по run-length, доповнення (total - bad), sliding window / queue

Ключові ідеї:
- Рахувати доповнення: `good = m^n - bad`, де `bad` — рядки без блоку ≥ k однакових підряд
- Prefix-state: важлива лише довжина хвоста однакових (run), обрізана до `k-1`
- “Bad → Good” стається тільки коли хвіст має `k-1` однакових і додаємо той самий колір (1 варіант)
- Хвости — це конвеєр, тому віднімання робиться через чергу з лагом `k-2`

Помилки:
- Замість “існує блок довжини k десь” я рахував “блок k на початку/в кінці” → пропускав випадки з блоком всередині
- Намагався “забороняти другу появу блоку” через страх дубліката, хоча кожен рядок у префіксному DP рахується рівно один раз
- Зламав інваріант черги: пушив не ту величину / переплутав, який елемент треба віднімати (зсув на 1) + офф-бай-ван у циклі довжини
Патерн:
- Для “є підряд k” → рахуй `bad` через run-length і роби `good = total - bad`; якщо стани зсуваються, тримай їх через queue/deque
